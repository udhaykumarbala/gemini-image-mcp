Generation

tool generate_image

Creates a new PNG image using Google Gemini. Accepts a plain-text prompt or a structured JSON prompt following the Nano Banana schema. Optionally applies a named preset that sets platform-appropriate dimensions. Reference images can be supplied for character/object consistency across a brand series.

Parameters
NameTypeRequiredDescription
prompt string required Natural language description OR stringified JSON prompt following the Nano Banana schema.
prompt_format "text" | "json" optional Whether the prompt is plain text or structured JSON. Default: text.
preset string optional Named preset (e.g. facebook_ad, hero_image). Sets aspect ratio and resolution; explicit params override.
aspect_ratio string optional Aspect ratio override. Examples: 16:9, 1:1, 9:16, 4:3, 21:9.
image_size "1K" | "2K" | "4K" optional Output resolution. Defaults to preset value or 1K.
model "flash" | "pro" optional flash = Gemini 3.1 Flash (fast/cheap). pro = Gemini 3 Pro (best quality). Default: flash.
reference_images string[] optional File paths to reference images. Loaded from disk and sent inline for character/object consistency.
output_name string optional Custom filename without extension. Auto-generated if omitted.
enable_search_grounding boolean optional Use Google Search grounding for real-world accuracy. Default: false.
Example Request
{
  "prompt": "Professional woman in a modern
office, warm golden hour lighting,
navy blazer, confident expression",
  "preset": "facebook_ad",
  "model": "flash",
  "output_name": "office-hero"
}
Example Response
{
  "image_path": "/output/office-hero.png",
  "model_used": "gemini-3.1-flash-image-preview",
  "aspect_ratio": "1:1",
  "resolution": "1K",
  "preset_applied": "facebook_ad",
  "prompt_used": "Professional woman in...",
  "description": "A professional woman..."
}

Analysis

tool decompose_image

Sends an existing image to Gemini and returns a structured JSON blueprint describing every visual component: subjects, scene, lighting, composition, camera settings, and style. The blueprint is cached as {image_name}.blueprint.json and is automatically used by edit_image for precise JSON edits.

Parameters
NameTypeRequiredDescription
image_path string required Absolute path to the image to decompose.
detail_level "basic" | "detailed" | "exhaustive" optional Controls decomposition depth. basic = ~10 fields. detailed = ~30 fields (default). exhaustive = ~60+ fields including camera, film stock, hex colors.
Example Request
{
  "image_path": "/output/hero.png",
  "detail_level": "detailed"
}
Example Response
{
  "image_path": "/output/hero.png",
  "blueprint": {
    "subject": [{
      "id": "person_1",
      "type": "person",
      "description": "Young woman, short black hair",
      "hair": { "style": "pixie_cut", "color": "jet_black" },
      "expression": "smiling",
      "clothing": [{
        "item": "blazer",
        "color": "#2C3E50",
        "fabric": "wool",
        "fit": "slim"
      }],
      "position": "center",
      "pose": "standing, arms crossed"
    }],
    "scene": {
      "location": "modern office with glass walls",
      "time": "golden_hour",
      "lighting": {
        "type": "natural_sunlight",
        "direction": "side_lit"
      }

Editing

tool edit_image

Edits an existing image using one of two modes. JSON mode accepts a changes object with dot-notation paths (e.g. subject[0].hair.color) and produces precise, field-level edits. Natural language mode accepts a plain-text instruction for complex or abstract scene transformations. The edited image is saved as {original_name}-edit-{n}.png.

Parameters
NameTypeRequiredDescription
image_path string required Absolute path to the image to edit.
edit_type "json" | "natural_language" required How the edit is described. Use json for precise field changes; natural_language for complex transformations.
changes object optional* Required when edit_type is "json". Object with dot-notation paths as keys and new values. E.g. { "subject[0].hair.color": "platinum_blonde" }.
instruction string optional* Required when edit_type is "natural_language". Plain-text edit instruction.
blueprint object optional Full JSON blueprint. If omitted, the cached .blueprint.json is used automatically.
preserve_style boolean optional Explicitly instructs the model to keep unchanged elements identical. Default: true.
model "flash" | "pro" optional Model to use. Default: flash.
output_name string optional Custom filename for the edited image. Auto-incremented if omitted.
JSON Edit — Example
Request
{
  "image_path": "/output/hero.png",
  "edit_type": "json",
  "changes": {
    "subject[0].hair.color": "platinum_blonde",
    "subject[0].accessories": [
      {
        "item": "sunglasses",
        "material": "metal",
        "color": "gold",
        "location": "face"
      }
    ]
  }
}
Response
{
  "original_path": "/output/hero.png",
  "edited_path": "/output/hero-edit-1.png",
  "edit_type": "json",
  "changes_applied": [
    "subject[0].hair.color",
    "subject[0].accessories"
  ],
  "model_used": "gemini-3.1-flash-image-preview",
  "description": "The image now shows..."
}
Natural Language Edit — Example
Request
{
  "image_path": "/output/hero.png",
  "edit_type": "natural_language",
  "instruction": "Change the background to a
tropical beach at sunset. Keep the
person exactly the same."
}
Response
{
  "original_path": "/output/hero.png",
  "edited_path": "/output/hero-edit-1.png",
  "edit_type": "natural_language",
  "model_used": "gemini-3.1-flash-image-preview",
  "description": "Background changed to..."
}

Catalog

tool get_presets

Returns all available asset presets. Each preset provides platform-appropriate dimensions, a recommended resolution, and usage tips. Pass the preset name to generate_image to apply its settings.

Parameters
NameTypeRequiredDescription
category "ad" | "web" | "social" | "print" | "all" optional Filter presets by category. Default: all. Available: ad (Facebook, Instagram, Google), web (hero, OG, product), social (LinkedIn, Twitter, YouTube), print.
Example Request
{
  "category": "ad"
}
Example Response
{
  "presets": [

      "name": "facebook_ad",

tool list_generated

Scans the output directory and returns metadata, blueprint availability, and edit-chain history for previously generated images. Supports keyword filtering by filename and optional inclusion of full blueprint JSON.

Parameters
NameTypeRequiredDescription
filter string optional Search term matched against filenames.
limit number optional Maximum number of results. Default: 20.
include_blueprints boolean optional When true, includes the full blueprint JSON for each image that has one. Default: false.
Example Request
{
  "filter": "hero",
  "limit": 10,
  "include_blueprints": true
}
Example Response
{
  "images": [
    {
      "path": "/output/hero.png",
      "name": "hero.png",
      "created_at": "2026-03-22T10:30:00Z",
      "size_bytes": 2048000,
      "has_blueprint": true,
      "edit_history": [
        "hero-edit-1.png",
        "hero-edit-2.png"
      ],
      "metadata": {
        "model": "flash",
        "prompt": "Professional woman in...",
        "prompt_format": "text",
        "preset": "hero_image",
        "aspect_ratio": "21:9",
        "image_size": "2K",
        "created_at": "2026-03-22T10:30:00Z"
      },
      "blueprint": {
        "subject": ["..."],
        "scene": "..."
      }
    }
  ],
  "total": 1
}

Workflow Examples

1
Generate New Asset
  1. 1
    Call get_presets(category: "ad") to browse platform presets and pick the right dimensions.
  2. 2
    Call generate_image(prompt: "...", preset: "facebook_ad"). The preset's aspect ratio and resolution are applied automatically.
  3. 3
    Review the returned image_path and open the generated PNG.
2
Edit Existing Asset (JSON — Precise Changes)
  1. 1
    Call decompose_image(image_path: "/output/hero.png") to get the full JSON blueprint of every visual component.
  2. 2
    Identify the dot-notation paths to change in the blueprint (e.g. subject[0].hair.color).
  3. 3
    Call edit_image(image_path: "/output/hero.png", edit_type: "json", changes: {"subject[0].hair.color": "platinum_blonde"}). The edited image is saved as hero-edit-1.png.
3
Edit Existing Asset (Natural Language — Complex Changes)
  1. 1
    Call edit_image(image_path: "/output/hero.png", edit_type: "natural_language", instruction: "Change the background to a tropical beach at sunset").
  2. 2
    If a cached blueprint exists it is included as context so unchanged elements are preserved more accurately.
  3. 3
    Review the returned edited_path. Iterate with additional natural language instructions if needed.
